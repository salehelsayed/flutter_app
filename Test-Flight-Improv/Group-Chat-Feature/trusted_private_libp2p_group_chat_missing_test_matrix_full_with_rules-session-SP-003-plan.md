## real scope

SP-003 covers random artifacts that are generated by the shipped trusted-private group stack:

- Go group AES-256 keys and AES-GCM nonces
- Go X25519 contact-request ephemeral public keys used as HKDF salts and AES-GCM nonces
- BIP39 identity entropy that drives libp2p Peer IDs and Ed25519 keys
- Go bridge group IDs and native publish message IDs
- Flutter group message IDs and direct invite IDs generated through locked `uuid` v4 defaults

The current product does not ship a separate public/link invite token generator. Direct invites use UUID v4 invite IDs, and push tokens are generated by platform push providers rather than locally.

## closure bar

SP-003 can be resolved when direct tests prove:

- random keys decode to expected byte lengths and do not collide across repeated samples
- nonces and ciphertexts change across repeated encryption of stable plaintext
- HKDF salt material derived from X25519 ephemeral public keys is fresh
- generated group IDs, native publish message IDs, Flutter message IDs, and direct invite IDs are UUID v4 and unique across repeated samples
- identity mnemonics and Peer IDs are unique across repeated generation
