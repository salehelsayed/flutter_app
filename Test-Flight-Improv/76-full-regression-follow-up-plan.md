## Final verdict

Session classification: `evidence-gated`.

The full-regression run in `.full_regression_logs/20260426_103445` is not a
reason to reopen Report 76's outbound plaintext removal. Current code already
enforces the Report 76 fail-closed contract: ordinary outbound 1:1 send/delete
needs a bridge plus recipient ML-KEM key, and legacy v1 retry shortcuts are
guarded.

The follow-up is safe only if it first separates stale test fixtures from real
product regressions:

- Stale fixture drift: tests still model keyless or legacy-v1 outbound success,
  or use fixed pending-introduction dates that now expire on 2026-04-26.

  still fails, drops ordering, or cannot deliver encrypted inbox/transport.
- Analyzer failure: one real analyzer error in a test fake plus a large
  pre-existing warning/info backlog. The one invalid override is in scope for
  this follow-up; the 1700 warning/info cleanup is a separate release hygiene
  blocker unless the project requires `flutter analyze` to be fully green.

### 12. known-failure interpretation

- A test that expects ordinary 1:1 send success without a bridge/key is stale.
- A test that expects v1 outbound replay as "valid" retry behavior is stale.
- A test that expects a pending introduction older than 30 days to stay pending
  is stale unless it explicitly freezes time or names the expiration behavior.
- `encryptionRequired` with no recipient ML-KEM key is expected behavior.
- `encryptionRequired` with a valid contact key is a product or wiring
  regression.

  stale key fixture. Treat it as product/evidence-gated until isolated.
- Existing analyzer warning/info backlog is not a Report 76 product regression,
  but `flutter analyze` cannot be reported green until the project either fixes
  it or adjusts analyzer policy.
