### Go Native Library Components

| Component | Responsibility |
|-----------|----------------|
| Bridge Entry (bridge/bridge.go) | Routes incoming requests to handlers via command dispatch; JSON request/response encoding; supports identity, crypto, P2P, inbox, media, profile, encrypt_contact_request, decrypt_contact_request, relay_reconnect, relay_probe, group:create/join/leave/publish/updateConfig/rotateKey/updateKey/inboxStore/inboxRetrieve, group.keygen/encrypt/decrypt commands |
| Identity Module (identity/) | `identity.generate` → BIP39 mnemonic + Ed25519 keypair + ML-KEM-768 keypair; `identity.restore` → derive from mnemonic + fresh ML-KEM-768 |
| Signing Module (bridge/bridge.go) | `payload.sign` → Ed25519 signing; `payload.verify` → Ed25519 signature verification |
| Crypto Module (crypto/) | `mlkem.keygen` → ML-KEM-768 keypair; `message.encrypt` → KEM encapsulate + AES-256-GCM; `message.decrypt` → KEM decapsulate + AES-256-GCM; `EncryptContactRequest()`, `DecryptContactRequest()` — X25519 ECDH + HKDF + AES-256-GCM |
| Group Crypto Module (crypto/group.go) | `group.keygen` → random AES-256 key (base64); `group.encrypt` → AES-256-GCM encrypt; `group.decrypt` → AES-256-GCM decrypt; GenerateGroupKey(), EncryptGroupMessage(), DecryptGroupMessage() |
| Node Module (node/) | libp2p node lifecycle (start/stop/status), circuit relay v2, NAT port mapping, hole punching |
| Rendezvous Module (node/rendezvous.go) | Peer registration and discovery with signed peer records via protobuf protocol |
| Chat Protocol (node/node.go) | `/mknoon/chat/1.0.0` — bidirectional message exchange with ACK replies, 4-byte big-endian length-prefixed frames |
| Inbox Module (node/inbox.go) | `/mknoon/inbox/1.0.0` — offline message store/retrieve/register FCM token via relay server |
| Media Module (node/media.go) | Media upload/download/delete/list and profile upload/download via relay `/mknoon/media/1.0.0` protocol |
| PubSub Module (node/pubsub.go) | GossipSub initialization with FloodPublish(true), JoinGroupTopic/LeaveGroupTopic, GroupPublish, topic validator (signature + key epoch verification), subscription handler goroutine, group peer discovery loop |
| Group Module (node/group.go) | GroupConfig, GroupKeyInfo, GroupMember structs; GroupType (chat/announcement/qa), GroupRole (admin/writer/reader) enums |
| Group Inbox Module (node/group_inbox.go) | GroupInboxStore/GroupInboxRetrieve for store-and-forward group messages via relay `/mknoon/inbox/1.0.0` protocol |
| Group Envelope Module (internal/group_envelope.go) | v3 group message wire format: GroupEnvelope (version, type, groupId, senderId, senderPublicKey, signature, keyEpoch, encrypted), GroupEncryptedPayload (ciphertext, nonce), GroupMessagePayload (text, timestamp, username, extra); BuildGroupEnvelope/ParseGroupEnvelope |
| X25519 Crypto (crypto/x25519.go) | X25519 ECDH + HKDF-SHA256 + AES-256-GCM for contact request encryption; Ed25519->X25519 key conversion; AAD = "msgId\|ts" |
| Testpeer CLI (cmd/testpeer/) | Headless Go CLI peer for E2E transport testing; stdin/stdout JSON protocol; V1/V2 envelope builder |
| Event System (node/node.go) | Push events to Flutter: `message:received`, `peer:connected`, `peer:disconnected`, `addresses:updated`, `group:message`, `group:discovery`, `group:publish_debug` via callback interface |
| Platform Wrappers | GoBridge.swift (iOS) + GoBridge.kt (Android) — bridge MethodChannel/EventChannel to Go library |
